VCS

A version control system, or VCS, tracks the history of changes as people and teams collaborate on projects together. As the project evolves, teams can run tests, fix bugs, and contribute new code with the confidence that any version can be recovered at any time.

Creating a repository

Initializing a git

  • To initialize a git, git commit is used

Adding files in the repository

  • To add files in a repository, git add is used. To add a single file, use "git add FileName.ext" where ext is the extension of the file or to add all files, use "git add .".

Commiting changes

  • git commit is used to commit changes and double quotes are used to describe the changes.

Pushing the changes onto the remote repository

  • git push is used to push the changes onto the remote repository

Status of the repository

  • git status is used to check the status of the repository

Branch

  • The git branch command lets you create, list, rename, and delete branches. You can also switch between branches using checkout.

Git Pull

  • git pull is used to make the changes in our local repository which were made at the online repository. Here I made changes in the online repository.

  • Then I used git pull to make those changes available in my local repository.
